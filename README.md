# base24-to-shadcn

Convert any [tinted-theming base24](https://github.com/tinted-theming/base24) color scheme into a [shadcn/ui](https://ui.shadcn.com) / [Basecoat UI](https://basecoatui.com) CSS theme.

## Quick Start

```bash
```bash
bun src/cli.ts <scheme.yaml> [-o theme.css] [-p base0D] [-r 0.625rem] [--preview]
```

Or via npm script:

```bash
bun run convert -- <scheme.yaml> [-o theme.css] [-p base0D] [-r 0.625rem] [--preview]
```
```bash
# Convert a local scheme
bun src/cli.ts ~/schemes/one-dark.yaml -o theme.css

# Convert from URL + generate preview HTML
bun src/cli.ts https://raw.githubusercontent.com/tinted-theming/schemes/spec-0.11/base24/one-dark.yaml -o theme.css --preview

# Use magenta as primary, custom border radius
bun src/cli.ts one-dark.yaml -p base0E -r 0.75rem
```

### Flags

| Flag | Default | Description |
|---|---|---|
| `-o, --output` | `./theme.css` | Output path for theme CSS |
| `-p, --primary` | `base0D` | Which base24 slot to use for primary/ring/chart-1 |
| `-r, --radius` | `0.625rem` | Border radius (used in preview HTML) |
| `--preview` | off | Also generate `preview.html` with Basecoat UI components |

## How It Works

### Color token mapping

The 24 base24 slots map to shadcn semantic tokens:

```
base00 ──→ background, primary-foreground
base01 ──→ card, popover, muted, sidebar-accent
base02 ──→ secondary, accent, sidebar-border
base03 ──→ border, input
base04 ──→ muted-foreground
base05 ──→ foreground, card-foreground, …
base0D ──→ primary, ring, chart-1      ← configurable via -p
base08 ──→ destructive, chart-5
base0A ──→ chart-3 (yellow)
base0B ──→ chart-2 (green)
base0E ──→ chart-4 (magenta)
base10 ──→ sidebar (falls back to base11, then base00)
```

### Mode detection

If the scheme YAML has a `variant: "dark"` or `variant: "light"` field, that's used. Otherwise, relative luminance of `base00` vs `base07` determines the mode.

- **Dark scheme** → real colors in `.dark { }`, auto-generated light variant in `:root { }`
- **Light scheme** → real colors in `:root { }`, auto-generated dark variant in `.dark { }`

The opposite variant swaps the neutral axis (base00↔base07, etc.), caps accent chroma, and adjusts lightness for readability.

### Output

The generated `theme.css` contains **only color tokens** — no radius, spacing, or shadows. Those belong to the Basecoat style pack (Vega, Lyra, etc.). Use with:

```css
@import "tailwindcss";
@import "basecoat-css/sera";
@import "./theme.css";
```

The file includes a commented-out `@theme inline` block for Tailwind v4 users who want color utilities like `bg-background`, `text-primary`.

Each CSS file has an attribution header:

```css
/*
 * base24 theme: One Dark
 * Author: FredHappyface (https://github.com/fredHappyface)
 * Source: https://github.com/tinted-theming/schemes/tree/spec-0.11/base24
 * Basecoat style pack: vega (https://basecoatui.com)
 * Generated by base24-to-shadcn
 * Only color tokens — radius, spacing, shadows belong to the style pack.
 */
```

## Live Explorer

Run the gallery + proxy server:

```bash
bun src/server.ts
```

### Pages

| URL | Description |
|---|---|
| `http://localhost:3000` | Gallery — browse all 190 base24 schemes with live preview |
| `http://localhost:3000/?scheme=one-dark` | Shareable link to a specific scheme |
| `http://localhost:3000/browse?scheme=one-dark` | Basecoat UI docs site proxied with theme injected |
| `http://localhost:3000/browse?scheme=one-dark&style=lyra` | Same, with Lyra style pack |
| `http://localhost:3000/browse/components/button/?scheme=one-dark` | Any basecoatui.com page with theme |
| `http://localhost:3000/theme.css?scheme=one-dark` | Raw downloadable CSS |
| `http://localhost:3000/theme.css?scheme=one-dark&style=lyra` | CSS with style pack attribution |

### Gallery Features

- **190 schemes** loaded from GitHub, searchable
- **Live preview** with Basecoat UI components (buttons, cards, inputs, badges, alerts, form)
- **Customize CSS Variables** — remap any token to any palette slot, changes apply instantly
- **Reset All** / per-row reset
- **CSS ↓** download button for the current theme (respects custom mappings)
- **Open in Basecoat ↗** — opens the theme on the live basecoatui.com docs
- **URL sharing** — scheme selection updates the URL, back/forward works
- **Palette swatches** with semantic role labels showing which CSS tokens each slot feeds
- **Primary slot selector** — change which slot drives `--primary`, `--ring`, `--chart-1`
- Scheme mode auto-detected (dark/light), no manual toggle needed

### Browse Page Features

- **Two independent dropdowns**: Style pack (Vega/Nova/Maia/Lyra/…) × Color theme (190 schemes)
- **Style pack** controls border-radius, shadows, hover effects — all visual design
- **Color theme** provides only the palette through CSS variables
- **Click any link** — navigation stays within the proxy, theme persists
- **"Default" option** — view basecoatui.com without any theme
- **CSS link** and **GitHub source link** in the floating picker

## Files

```
src/cli.ts      CLI: single-scheme converter, outputs theme.css + preview.html
src/server.ts       Bun HTTP server: gallery + basecoatui.com proxy + CSS API
gallery.html    Self-contained browser gallery (served by src/server.ts)
```

## Architecture

### src/cli.ts (CLI)

- **No dependencies** — self-contained TypeScript, runnable with `bun`
- Parses YAML inline (flat + nested formats, inline comments)
- Full Björn Ottosson hex→oklch conversion (D65, gamma 2.2)
- Generates standalone `theme.css` and optional `preview.html`

### src/server.ts

- **Proxy mode** — fetches basecoatui.com pages, strips `hx-boost`, injects theme CSS + floating picker
- **CSS API** — `/theme.css?scheme=name` returns raw CSS with attribution header
- **Scheme API** — `/schemes` returns JSON list of all schemes
- In-memory cache for schemes and generated CSS
- Supports `?style=` param to attribute the style pack in CSS comments

### gallery.html

- All conversion logic ported to browser JavaScript
- Fetches scheme list from GitHub API, individual YAML from raw URLs
- Dynamic theme generation and live injection into `#theme-style`
- Custom CSS variable remapping with instant preview

## Design Decisions

- **`--radius` excluded from theme output** — it's a visual design concern, belongs to the style pack
- **Only color tokens output** — 24 CSS custom properties, all color-related
- **Opposite variant auto-generated** — every scheme produces both light and dark blocks
- **Cascade fixes for Tailwind v3** — the gallery uses Tailwind v3 CDN which conflicts with Basecoat v4's `@layer`; explicit unlayered rules fix this
- **CDN via unpkg** — jsDelivr had 503 for freshly published `basecoat-css@1.0.1`
